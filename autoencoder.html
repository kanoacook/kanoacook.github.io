Anomaly Prediction for UF/XMF Filtration

At Amgen, the filtration step of monoclonal antibodies is monitored in real-time using PI and PI Historian, the yield is measured using an MES system, and any sort of equipment maintenence is recorded in Maximo -- getting these systems to talk to each other requires some data engineering!

Data Engineering & Visualization

I leveraged Databricks to create an automated data pipeline to fetch information from the source systems using their APIs and store the data into AWS Redshift. We built a webhook such that the data was only fetched when new data was created -- instead of trying to pull the data on a scheduled cadence. Joining all of the data together in one dashboard Spotfire, allowed for our user group to have all of the relevant information in one spot. Score! However, we decided to go a step further.

We created two separate ML functions (slow-feature analysis and an autoencoder) to detect any variations from ideal operation. My co-worker developed a SFA algorithm that would take user-inputted baselines to act as training group for the algorithm that serves as the "ideal" operations. I created a web application for this model selection in Dash. Additionally, I developed an autoencoder that would be able to predict a bad batch without user-inputted baselines to eliminate bias!
